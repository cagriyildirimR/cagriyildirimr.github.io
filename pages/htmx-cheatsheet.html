HTMX Cheat Sheet

Introduction

HTMX is a library that allows you to access modern browser features directly from HTML, rather than using JavaScript. This cheat sheet covers the core attributes and patterns used in this website, helping you quickly reference the HTMX features as you explore the code.

HTMX version used in this site: 1.9.10

Core HTMX Attributes

Attribute Description Example
hx-get Issues a GET request to the given URL <button hx-get="/api/data">Get Data</button>
hx-post Issues a POST request to the given URL <button hx-post="/api/submit">Submit</button>
hx-put Issues a PUT request to the given URL <button hx-put="/api/update">Update</button>
hx-delete Issues a DELETE request to the given URL <button hx-delete="/api/remove">Delete</button>
hx-patch Issues a PATCH request to the given URL <button hx-patch="/api/partial-update">Update Partially</button>
hx-target Specifies the target element to swap content into <button hx-get="/api/data" hx-target="#result">Load</button>
hx-swap Controls how the response is swapped into the DOM <button hx-get="/api/data" hx-swap="outerHTML">Replace</button>
hx-trigger Specifies what triggers the request <div hx-get="/api/data" hx-trigger="mouseenter">Hover Me</div>

Common Patterns

Click to Load Content

<button 
  hx-get="/api/data" 
  hx-target="#result" 
  class="bg-blue-500 text-white px-4 py-2 rounded"
>
  Load Data
</button>

<div id="result" class="mt-4">
  Result will appear here...
</div>

Form Submission

<form 
  hx-post="/api/submit" 
  hx-target="#form-result" 
  hx-swap="outerHTML"
>
  <input type="text" name="name" placeholder="Your name" class="border p-2 rounded">
  <button type="submit" class="bg-green-500 text-white px-4 py-2 rounded">
    Submit
  </button>
</form>

<div id="form-result"></div>

Automatic Load on Page Load

<div 
  hx-get="/api/initial-data" 
  hx-trigger="load"
  hx-swap="innerHTML"
  class="p-4 border rounded"
>
  Loading...
</div>

Load Content After Delay

<div 
  hx-get="/api/delayed-data" 
  hx-trigger="load delay:2s"
  class="p-4 border rounded"
>
  Content will load after 2 seconds...
</div>

Polling for Updates

<div 
  hx-get="/api/updates" 
  hx-trigger="every 5s"
  class="p-4 border rounded bg-gray-50"
>
  This content will refresh every 5 seconds
</div>

Swap Methods

Method Description
innerHTML Default. Replaces the inner HTML of the target element
outerHTML Replaces the entire target element with the response
beforebegin Inserts the response before the target element
afterbegin Inserts the response before the first child of the target element
beforeend Inserts the response after the last child of the target element
afterend Inserts the response after the target element
none Does not append content from response (out of band items will still be processed)

Integrating HTMX with Tailwind

This website uses both HTMX and Tailwind CSS. Here are some patterns for how they work together: 

<!-- Button with loading state using Tailwind classes -->
<button 
  hx-get="/api/data" 
  hx-target="#result"
  class="bg-blue-500 hover:bg-blue-600 text-white px-4 py-2 rounded transition-colors"
  hx-indicator="#spinner"
>
  Load Data
</button>

<!-- Hidden spinner that appears during loading -->
<div id="spinner" class="htmx-indicator">
  <div class="animate-spin h-5 w-5 border-4 border-blue-500 rounded-full border-t-transparent"></div>
</div>

<!-- Target element with Tailwind styling -->
<div id="result" class="mt-4 p-4 border rounded bg-white shadow-sm">
  Result will appear here...
</div>

CSS for handling the loading indicator: 

/* Add this to your CSS */
.htmx-indicator {
  opacity: 0;
  transition: opacity 200ms ease-in;
}
.htmx-request .htmx-indicator {
  opacity: 1;
}
.htmx-request.htmx-indicator {
  opacity: 1;
}

HTMX Events

HTMX provides a rich set of events that you can listen for: 

// Listen for successful responses
document.body.addEventListener('htmx:afterOnLoad', function(event) {
  console.log('Content loaded successfully!', event.detail.elt);
});

// Listen for errors
document.body.addEventListener('htmx:responseError', function(event) {
  console.error('Error loading content!', event.detail.xhr.status);
});

// Do something before a request is made
document.body.addEventListener('htmx:beforeRequest', function(event) {
  // You could show a global loading indicator here
  console.log('Request is about to be made');
});
